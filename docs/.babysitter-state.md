### Spec Contents

#### `src/lib/__tests__/provider-configs-storage.spec.ts`

Tests `getProviderConfigs` and `setProviderConfigs` using the existing `chrome.storage.local` mock pattern. Covers: empty-key returns `{}`, stored map round-trip, correct storage key used, multi-provider map, empty-map write, and the full set/get round-trip.

#### `src/sidepanel/__tests__/provider-configs-store.spec.ts`

Behavioral tests for the settings store using `vi.mock('../../lib/storage')`. Covers: `loadSettings` seeding `providerConfigs` from active provider when storage is empty; `loadSettings` restoring stored map when present; `saveSettings` merging new config into existing map; `saveSettings` calling `setProviderConfigs` with merged result.

#### `src/sidepanel/__tests__/settings-store.spec.ts` (additions)

Structural (file-read) checks added for: `providerConfigs` export, `getProviderConfigs` import, `setProviderConfigs` import, `loadSettings` referencing `getProviderConfigs`, `saveSettings` referencing `setProviderConfigs`.
